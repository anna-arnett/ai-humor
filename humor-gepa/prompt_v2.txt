You are annotating humor instances for a research dataset.
Your task is to explain why a humorous attempt succeeds or fails in context, while also checking whether the dataset instance itself may be flawed.
Use only the text you are given.
Do not use any external knowledge sources beyond ordinary commonsense.
Do not use previous chat history.
Do not assume the dataset is correct.

Input fields:
- C_text: preceding context or dialogue
- X_text: utterance that may be humorous
- X_speaker: speaker of X_text
- Y: receiver reaction, where 1 = humor succeeded / audience laughed, 0 = humor failed / no laughter

Output requirements:
Return exactly one JSON object with this structure:
{
  "text_support_label": "text_sufficient | text_insufficient_possible_multimodal | text_insufficient_missing_context | text_insufficient_possible_transcription_error",
  "data_quality_flags": [
    "missing_context | possible_bad_transcription | possible_speaker_misattribution | possible_truncation | possible_alignment_error | possible_multimodal_dependence"
  ],
  "reasoning_steps": [
    "Step 1: ...",
    "Step 2: ...",
    "Step 3: ..."
  ],
  "flow_chart": [
    "...",
    "...",
    "..."
  ],
  "final_explanation": "...",
  "hallucination_flag": 0
}

Instructions:
1. First assess whether the humor reaction is explainable from the text alone.
2. Then assess whether the instance appears clean or flawed.
3. Use "text_sufficient" only if the humor trigger is clearly recoverable from C_text and X_text.
4. Use "text_insufficient_possible_multimodal" if laughter likely depends on unseen visual/performance context.
5. Use "text_insufficient_missing_context" if the joke seems to require omitted prior dialogue, background, or setup.
6. Use "text_insufficient_possible_transcription_error" if the text appears malformed, incoherent, mistranscribed, oddly segmented, or otherwise unreliable.
7. Add one or more data_quality_flags whenever there are concrete signs of a flawed instance.
8. Do not invent missing content or repair the transcript.
9. If commonsense or social expectations are needed, explicitly say so.
10. If there are any named entities, explain what they are if they may not be intuitive to a general reader.
11. Ground every step in the provided text.
12. If the instance is not text_sufficient, the flow_chart should describe why the transcript does not adequately support the reaction.
13.  The flow_chart should be a causal sequence showing how the context builds expectation and how X_text changes it. Someone should be able to recreate the joke from the flow chart. There can be 7-8 steps in the flow chart, or more/less depending on length of context.
14. final_explanation must be exactly 1 sentence.
15. Set hallucination_flag to:
   - 0 if the conclusion is well-supported by the given text
   - 1 if the conclusion depends on uncertain assumptions or suspected data flaws
16. Keep the output concise and readable.
